# Support & feedback

mail-index has **no telemetry** — nothing is ever sent automatically, so the only
way we hear from you is if you reach out. All of these are opt-in and public
(except security, which is private):

| I want to… | Go here |
|---|---|
| **Ask a question / share how I use it / float an idea** | [Discussions](https://github.com/alunsoldantarctica/mail-index/discussions) |
| **Report a bug** | [New bug report](https://github.com/alunsoldantarctica/mail-index/issues/new?template=bug_report.yml) |
| **Request a feature / give feedback** | [New feature request](https://github.com/alunsoldantarctica/mail-index/issues/new?template=feature_request.yml) |
| **Report a security or privacy issue** (private) | [SECURITY.md](SECURITY.md) |
| **Reach the maintainer** | [unsold.group/al](https://unsold.group/al) |

Because there's no usage data on our end, **details help** — version, OS, Node
version, and steps make a bug actionable. Please **redact real email content,
addresses, and secrets** in anything you paste.

From inside your agent you can also just say *"report a mail-index bug"* or
*"send feedback about mail-index"* — the MCP server tells the agent there's no
auto-reporting and to hand you a GitHub link to submit yourself.
